# Security policy for AI Content Auditor (`ai_content_audit`)

## Supported versions

Security updates are intended for the latest stable release line once the
project is published on **Drupal.org**. Until then, treat this repository as
**pre-release**: pin deployments to explicit tags or commits and monitor changes.

## Reporting a vulnerability

**Before Drupal.org publication** — Contact the maintainers through the
private channel agreed for this repository (for example maintainer e-mail or
your organisation’s security inbox). Include:

- Affected version or Git commit
- Steps to reproduce
- Potential impact (confidentiality, integrity, availability)

**After Drupal.org publication** — Use the [Drupal.org security reporting process](https://www.drupal.org/drupal-security-team/security-team-procedures)
for the `ai_content_audit` project so the Drupal Security Team can coordinate
an advisory if appropriate.

## Scope

Reports should concern this module and its shipped submodules (for example
`ai_site_audit`). Third-party provider SDKs or the **Drupal AI** module itself
should be reported to their respective maintainers unless the issue is clearly
caused by integration code in `ai_content_audit`.

## Response expectations

Pre-release projects may not offer SLA-based response times. Commercial
support, if any, is outside the scope of this document.
